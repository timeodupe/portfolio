Résumé du projet

Le projet

Ce projet, réalisé en binôme dans le cadre du BTS SIO, vise à automatiser la création et la gestion sécurisée de comptes utilisateurs et de leurs bases de données associées. Il consiste à extraire des données utilisateurs depuis un Active Directory, à normaliser ces données, à générer des mots de passe robustes via Python, et enfin à produire des scripts SQL pour la création (et la suppression) massive de bases de données avec des privilèges cloisonnés.

Les enjeux

L'objectif principal était de garantir que chaque utilisateur dispose d'un espace de base de données personnel, entièrement cloisonné et sécurisé, tout en automatisant les tâches répétitives d'administration système pour éviter les erreurs humaines.

L'environnement technique

Le projet s'est déroulé dans un environnement virtualisé simulant un réseau d'entreprise complet (Domaine stesio.jol). L'infrastructure comprenait un serveur sous Rocky Linux 9.5 (MariaDB), un client Windows 10 Pro (VS Code / PowerShell), le tout sécurisé via une segmentation par VLAN et une authentification via Stormshield.

// realisations

Principales réalisations

🌐

1. Préparation de l'infrastructure

Configuration réseau des machines virtuelles (attribution d'IP statiques, plan de nommage) et jonction du poste Windows 10 au domaine Active Directory pour permettre l'administration centralisée du parc.

📥

2. Extraction et traitement

Développement du script PowerShell extractUsersPerpignan.ps1 pour récupérer les utilisateurs de l'AD vers un CSV. Utilisation de Python (unicodedata) pour normaliser les données et créer des logins standardisés (ex: pnom) sans accents ni caractères spéciaux.

🔐

3. Sécurité et Mots de passe

Développement d'un générateur Python (testPassword.py) créant des mots de passe ultra-sécurisés de 12 caractères minimum (majuscules, minuscules, chiffres, caractères spéciaux), couplé à une vérification anti-doublon.

⚙️

4. Automatisation SQL

Génération dynamique de scripts SQL via Python pour assurer que chaque utilisateur ne peut accéder qu'à sa propre base de données. L'automatisation prend en charge la création de la base, de l'utilisateur et l'attribution ciblée des droits :

script_generation.sql 
CREATE DATABASE IF NOT EXISTS st_utilisateur;
CREATE USER IF NOT EXISTS 'utilisateur'@'localhost' IDENTIFIED BY 'password';
GRANT CREATE, ALTER, DROP, INSERT, UPDATE, DELETE, SELECT ON st_utilisateur.* TO 'cauvray'@'localhost';
