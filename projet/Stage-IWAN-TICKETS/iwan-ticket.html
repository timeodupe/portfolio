Résumé du projet

Pourquoi ce projet ?

IWAN est une SARL spécialisée dans le développement de logiciels sur mesure (ERP, logiciel de caisse, réservation…). Leurs clients rencontraient des bugs et avaient des demandes d'évolution, mais sans outil centralisé pour les gérer. J'ai été missionné pour concevoir de zéro une application web de ticketing : chaque incident ou demande devient un ticket numéroté, priorisé et traçable, accessible directement depuis l'espace client existant.

Ce que j'ai réalisé

Conception complète de la base de données (dictionnaire de données → MLD sur Looping), développement du back-end en PHP natif avec architecture MVC, interfaces dynamiques avec requêtes asynchrones (AJAX / Fetch), intégration de l'API OpenAI, génération de PDF, envoi d'e-mails automatiques et déploiement sur serveur distant IONOS via FileZilla.

Résultat

Application entièrement fonctionnelle et mise en production à l'issue du stage. Deux interfaces distinctes : une pour les clients (création et suivi de tickets) et une pour l'équipe IWAN (gestion, réponses, suivis d'appels). La maquette Figma réalisée en amont par un autre stagiaire a servi de guide visuel tout au long du développement.

// features

Fonctionnalités clés

🎫

Gestion des tickets

Création, classification par niveau d'urgence, numérotation automatique et centralisation de toutes les demandes et incidents clients au même endroit.

Cœur métier
⚡

Interfaces dynamiques (AJAX)

Recherche d'entreprises en temps réel avec un délai de 300 ms, réponse JSON sans rechargement de page. Les données déjà saisies sont préservées pendant la recherche.

JavaScript / Fetch
📎

Pièces jointes intelligentes

Nommage unique via uniqid(), icônes SVG auto-générées selon l'extension, ouverture dans un nouvel onglet pour les fichiers compatibles navigateur ou téléchargement forcé pour les autres.

PHP · SVG
📄

Export PDF

Génération d'un ticket complet au format A4 imprimable via la librairie PHP Dompdf, installée avec Composer. Vue HTML/CSS dédiée pour un rendu propre et professionnel.

Dompdf · Composer
🤖

Correction IA (OpenAI)

Reformulation et correction orthographique des descriptions de tickets via l'API GPT. L'utilisateur prévisualise le texte modifié dans une modale avant de choisir de l'appliquer ou non.

API OpenAI
📧

E-mails automatiques

Notifications client et IWAN à chaque événement (nouveau ticket, réponse, suivi). Envoi via PHPMailer en SMTPS chiffré, templates HTML professionnels, gestion des erreurs en try/catch.

PHPMailer · SMTP
📋

Suivis d'appels

Section dédiée à la création de comptes-rendus d'appels liés aux clients, permettant à IWAN d'assurer une traçabilité complète de toutes les interactions, au-delà des seuls tickets.

Traçabilité
🚀

Déploiement en production

Transfert du code source via FileZilla (FTP), import de la base de données MySQL sur le serveur distant IONOS, passage d'un environnement local XAMPP à une application accessible en ligne.

FTP · Hébergement
